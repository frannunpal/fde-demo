# Internationalization (i18n)

This project supports multiple languages using `react-i18next`. Currently supported languages are English (default) and Spanish.

## Architecture

### Configuration

The i18n configuration is in `src/Infrastructure/i18n.ts`:

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';

// Import translation files
import enCommon from '@shared/Locales/en/common.json';
// ... other namespaces

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: { en: {...}, es: {...} },
    fallbackLng: 'en',
    supportedLngs: ['en', 'es'],
    // ...
  });
```

### Translation Files

Translations are organized by language and namespace:

```
src/Shared/Locales/
├── en/
│   ├── common.json      # Common UI strings (buttons, labels, themes)
│   ├── apps.json        # App names, descriptions, and folder names (folders.games, folders.office, etc.)
│   ├── menuedit.json    # MenuEditApp strings (predefined, customFolders, availableApps, etc.)
│   ├── settings.json    # Settings sections and options
│   ├── window.json      # Window-related strings
│   ├── contextMenu.json # Context menu strings
│   ├── notifications.json # Notification messages
│   ├── welcome.json       # Welcome app strings
└── es/
    └── ...              # Same structure
```

### Use the useTranslation Hook

```typescript
import { useTranslation } from 'react-i18next';

const MyComponent = () => {
  const { t, i18n } = useTranslation();

  return (
    <div>
      <h1>{t('common:labels.title')}</h1>
      <p>Current language: {i18n.language}</p>
      <button onClick={() => i18n.changeLanguage('es')}>
        Switch to Spanish
      </button>
    </div>
  );
};
```

## Language Detection

1. **First visit**: The app detects the browser's language using `navigator.language`. If it's a supported language (en/es), it's used. Otherwise, English is the default.
2. **Fallback**: If the detected language is not supported, English is used
3. **Persistence**: User's language preference is saved to localStorage with key `fde-desktop:language` (used by i18next-browser-languagedetector)
4. **Manual switch**: Users can change language via:
   - Settings app → Language section
   - Taskbar language selector (flag button)

## Language Synchronization

The `useLanguageSync` hook ensures the Zustand store and i18next stay synchronized:

```typescript
import { useLanguageSync } from '@presentation/Hooks/useLanguageSync';

// In your root component (App.tsx)
function App() {
  useLanguageSync(); // Keeps store and i18n in sync
  // ...
}
```

**How it works:**

1. On mount, it checks if the store's language differs from i18next's current language
2. If they differ, it calls `i18n.changeLanguage()` to sync them
3. It also saves the language to `localStorage` under `fde-desktop:language` for persistence across sessions

**Location**: `packages/core/src/hooks/useLanguageSync.ts`

## Adding a New Language

1. Create a new folder in `packages/core/src/i18n/locales/` (e.g., `fr/`)
2. Copy all JSON files from `en/` to the new folder
3. Translate all strings in the JSON files
4. Update `packages/core/src/i18n/index.ts`:

   ```typescript
   import frCommon from './locales/fr/common.json';
   // ... import other namespaces

   export const SUPPORTED_LANGUAGES = ['en', 'es', 'fr'] as const;

   // Add to resources:
   fr: {
     common: frCommon,
     // ...
   }
   ```

5. Update `LanguageSettings.tsx` and `LanguageSelector.tsx` with the new language flag and name

## Translation Keys Convention

Use nested keys organized by feature:

```json
{
  "folders": {
    "games": "Games",
    "office": "Office and media",
    "programming": "Programming",
    "socialLinks": "Social Links"
  },
  "actions": {
    "save": "Save",
    "cancel": "Cancel"
  },
  "labels": {
    "name": "Name",
    "title": "Title"
  },
  "messages": {
    "areYouSure": "Are you sure?",
    "cannotBeUndone": "This action cannot be undone."
  }
}
```

Access them with: `t('namespace:key.path')` or `t('key.path')` if using default namespace.

## Interpolation

Use double curly braces for dynamic values:

{% raw %}

```json
{
  "messages": {
    "replaceFileMessage": "A file named \"{{name}}\" already exists."
  }
}
```

{% endraw %}

```typescript
t('messages.replaceFileMessage', { name: 'document.txt' });
```

## Testing with i18n

The test setup includes mocks for both `react-i18next` and `i18next` in `src/Shared/Testing/__mocks__/jsdom-setup.tsx`. The mock returns translation keys as values with namespace and interpolation support:

{% raw %}

```typescript
// The mock handles namespaces and interpolation
vi.mock('react-i18next', () => ({
  useTranslation: (defaultNs?: string) => ({
    t: (key: string, options?: Record<string, unknown>) => {
      if (options?.defaultValue) return options.defaultValue as string;
      const ns = (options?.ns as string | undefined) ?? defaultNs;
      let result = key;
      if (ns && !result.includes(':')) {
        result = `${ns}:${result}`;
      }
      // Interpolation: {{name}} -> value
      if (options) {
        Object.entries(options).forEach(([k, v]) => {
          if (k !== 'ns' && k !== 'defaultValue') {
            result = result.replace(new RegExp(`{{\\s*${k}\\s*}}`, 'g'), String(v));
          }
        });
      }
      return result;
    },
    i18n: { changeLanguage: vi.fn(), language: 'en' },
  }),
  Trans: ({ children }) => children,
  initReactI18next: { type: '3rdParty', init: vi.fn() },
}));
```

{% endraw %}

Tests should use translation keys instead of hardcoded strings:

```typescript
// Good: using translation key
expect(screen.getByLabelText('window.ariaLabels.close')).toBeInTheDocument();

// Good: using role-based queries with translation keys
expect(screen.getByRole('button', { name: 'common:actions.save' })).toBeInTheDocument();

// Good: with interpolation
expect(screen.getByText('messages.replaceFileMessage: {{name}}test.txt')).toBeInTheDocument();
```

For files that import `i18next` directly (e.g., menu bar builders), the mock is also provided:

```typescript
vi.mock('i18next', () => ({
  default: {
    t: (key: string) => key,
    use: vi.fn(() => mockI18n),
    init: vi.fn(() => mockI18n),
    changeLanguage: vi.fn(),
    language: 'en',
  },
}));
```
